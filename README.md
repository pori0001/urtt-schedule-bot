# 🎓 URTT Schedule Bot (Telegram)

Асинхронный Telegram-бот для парсинга и удобного просмотра расписания занятий Уральского радиотехнического колледжа им. А.С. Попова (УРТТ).

Бот разработан с упором на производительность, отказоустойчивость и удобство пользователя. Он не просто парсит страницу по запросу, а использует **Data-Driven подход**: фоновый воркер собирает данные всего колледжа в локальную базу данных, а бот мгновенно отдает их пользователям.

## ✨ Основные возможности

- **Мгновенный ответ**: Данные берутся из локальной БД (SQLite), а не запрашиваются с сайта при каждом клике.
- **Умная нормализация ввода**: Бот понимает хаотичный ввод групп (`Ис - 326`, `is326`, `ИС-326`) и приводит его к единому стандарту.
- **Вычисление совмещенных пар**: Бот автоматически анализирует БД и показывает, с какими группами пересекаются пары в одном кабинете (реализовано через SQL `GROUP_CONCAT`).
- **Поиск по кабинету**: Возможность посмотреть, какие группы и когда занимают определенный кабинет в текущий день.
- **Система уведомлений (Polling)**: Фоновый процесс каждые 5 минут сверяет текущее расписание на сайте с базой данных. При обнаружении изменений бот точечно рассылает уведомления студентам затронутых групп.
- **Учет часовых поясов**: Корректная работа с датами и временем (UTC+5, Екатеринбург) независимо от физического расположения сервера.

## 🛠 Стек технологий

- **Язык**: Python 3.10+
- **Фреймворк для Telegram**: `aiogram 3.x` (Асинхронный)
- **База данных**: `SQLite` (асинхронное взаимодействие через `aiosqlite`)
- **Парсинг HTML**: `BeautifulSoup4`
- **Асинхронные HTTP-запросы**: `aiohttp` (Пул воркеров через `asyncio.Semaphore` для защиты сайта от DDoS)
- **Планировщик задач**: `APScheduler`
- **Работа со временем**: `pytz`, `datetime`

## 🏗 Архитектура

Приложение построено по принципу "Монолит с фоновым воркером":

1.  **Parser (Фоновая задача)**: 
    Раз в 5 минут асинхронно скачивает расписание всех 80+ групп колледжа. Для предотвращения блокировки со стороны сайта колледжа (HTTP 429) используется семафор, ограничивающий количество одновременных подключений до 15.
2.  **Database (SQLite)**: 
    Хранит актуальный слепок расписания всего учебного заведения и настройки пользователей. Индексы (`CREATE INDEX`) обеспечивают мгновенный поиск по кабинетам и группам.
3.  **Telegram Handler**: 
    Обрабатывает команды пользователей и формирует красивые ответы, обращаясь исключительно к локальной базе данных.
